# MIearn V2 发音、两遍学习与记忆系统设计

日期：2026-06-27  
状态：待用户书面复核

## 1. 目标与原则

MIearn V2 首先修复发音静默失效，再把新词学习改为低负担的“两遍学习”，随后
加入 SM-2 复习、错题强化、学习统计、每日提醒和记忆曲线。

产品遵循以下原则：

- 打开应用后，用户能够用拇指自然点击一次就开始当天任务。
- 新词第一次出现时只浏览，不要求用户判断“认识、模糊或不认识”。
- 复习质量由客观答题表现计算，避免额外的自我评分。
- 首页保留必要数据，但不加入资讯流、排行榜、活动 Banner 或运营入口。
- 核心学习和发音在断网状态下可用。

## 2. 信息架构与首页

底部导航由三个入口缩减为两个：

1. **学习**：合并原首页和词库页。
2. **测试**：保留五种独立测试模式。

“学习”首页按照视觉优先级排列：

- 顶部：MIearn、当前词库名称和右上角设置。
- 今日摘要：新学数量、到期复习数量、连续学习天数。
- 次级入口：切换词库、搜索、收藏、错题和学习数据。
- 主操作：大号“开始学习”按钮固定在底部导航上方 16–24dp，横向留出安全边距，
  高度至少 56dp，始终处于单手拇指舒适区。

详细数据页、设置页和学习任务页作为沉浸式二级页面，不占用底部导航。

## 3. 每日学习状态机

点击“开始学习”后直接进入任务，不显示额外配置页。任务由三个阶段组成：

### 3.1 到期复习

- 展示英文和四个中文选项。
- 每张卡出现时自动发音。
- 首次答对记为 SM-2 质量 5。
- 首次答错记为 SM-2 质量 2，显示正确答案，加入本轮末尾强化队列并增加错题次数。
- 末尾强化只帮助即时记忆，不会再次修改本次 SM-2 计划或重复增加学习统计。

### 3.2 新词浏览

- 按词库原顺序展示当天新词。
- 每张卡出现时自动发音。
- 左右滑动切换词条；点击卡片在精简态和完整态之间切换。
- 精简态突出英文；完整态显示音标、中文、双语例句和备注。
- 发音与收藏按钮始终可用。
- 本阶段不显示熟悉度按钮，不更新 SM-2 评分。

### 3.3 新词巩固

- 再次展示同一批新词，题型固定为“英文选择中文”。
- 每张题卡出现时自动发音。
- 首次答对记质量 5，并进入次日复习。
- 首次答错记质量 2，显示正确答案，加入本轮末尾强化队列、错题本和次日优先复习。
- 强化阶段答对后继续；无论强化结果如何，都不覆盖首次答案。

任务完成后只展示学习数、复习数、首次正确率和待复习时间。学习阶段、词条顺序、
当前位置、已展开状态和强化队列写入本地；应用被杀死后可以继续。

## 4. 发音架构

### 4.1 已定位的 V1 根因

V1 APK 没有打包词条音频，因此运行时完全依赖系统 TTS。`TextToSpeech` 初始化是
异步的，但 V1 在初始化完成前收到播放请求时只检查 `ttsReady`，随后静默丢弃
请求；初始化完成后不会补播。代码也没有播放错误监听、英语语音选择、失败重试或
缺少语音数据时的用户引导。

### 4.2 V2 播放策略

每个词条打包一个 `audio/<wordId>.ogg`，运行时使用以下优先级：

1. Media3 播放内置 Ogg。
2. 内置文件不存在或播放器报错时，回退 Android TTS。
3. TTS 不可用时展示简短状态和“安装/启用英语语音”系统入口。

内置音频为单声道、32kbps Ogg/Opus；生成前输入重采样为 22.05kHz。Opus 解码端
按标准报告 48kHz。构建校验要求 2,704 个文件全部存在，并对全部文件执行解码
检查。

### 4.3 TTS 兼容层

发音组件暴露 `Initializing`、`Ready`、`Speaking` 和 `Unavailable` 状态：

- 初始化期间只保留最新播放请求；初始化成功后自动补播。
- 优先选择本地 `en-US` voice，找不到时回退任意本地英语 voice。
- 设置英语区域、适中的语速和音高，并请求媒体音频焦点。
- 使用 `UtteranceProgressListener` 记录开始、完成和错误。
- TTS 播放错误自动重试一次，第二次失败后进入 `Unavailable`。
- 快速滑动会停止上一词条，使用请求序号保证过期回调不能覆盖当前状态。
- 手动点击发音始终允许重播当前卡。

内置音频提供跨厂商的基础保证；TTS 负责资源异常和未来新增词条，不把系统语音包
作为唯一可用链路。

## 5. SM-2 与错题强化

`ProgressEntity` 升级后增加：

- `easeFactor`，初始 2.5，最低 1.3。
- `intervalDays`。
- `repetitions`。
- `lapseCount`。
- `nextReviewEpochDay`。
- `lastReviewedEpochDay`。
- `firstLearnedEpochDay`。

SM-2 更新公式：

```text
EF' = max(1.3, EF + 0.1 - (5-q) * (0.08 + (5-q) * 0.02))
```

- `q < 3`：`repetitions = 0`、`intervalDays = 1`、遗忘次数加一。
- 第一次成功：间隔 1 天。
- 第二次连续成功：间隔 6 天。
- 后续成功：`round(previousInterval * easeFactor)` 天。
- `mastered` 在连续成功至少三次且间隔达到 21 天后置为真。

错题强化遵循两个层级：

- 本轮强化：首次答错的题在当前任务末尾再次出现。
- 跨日强化：`wrongCount > 0` 的词条在到期复习中优先排序；后续首次答对会逐次降低
  错题权重，连续正确后退出错题入口。

## 6. 数据模型与迁移

数据库升级时保留 V1 的收藏、错题、已学状态、连续认识次数和最后学习日期。

新增 `ReviewEventEntity`，每次首次作答保存：

- 词条 ID、分类、事件时间和日期。
- 阶段：到期复习或新词巩固。
- 首次是否正确、SM-2 质量、作答耗时。
- 更新后的间隔与到期日期。

新增单例 `StudySessionEntity` 保存：

- 当前词库、任务日期和阶段。
- 复习 ID、新词 ID、当前索引和强化 ID。
- 已完成计数、首次正确计数和当前卡展开状态。

任务正常结束后删除会话；跨日恢复时先提示用户继续昨日任务或开始今日任务，避免把
昨日答题计入今日活动。

## 7. 学习统计与记忆曲线

首页只显示今日新学、今日复习、连续天数和总进度。点击“学习数据”进入详情页：

- 近 7 天与近 30 天学习/复习柱状趋势。
- 首次正确率。
- 新学、复习、错题和已掌握总量。
- 未来 7 天预计到期复习数量。
- 当前词库的预计保持率曲线。

预计保持率根据最近复习后经过天数和当前 SM-2 间隔计算：

```text
retention = exp(-elapsedDays / max(1, intervalDays))
```

曲线明确标注为“预计保持率”，不作为医学或认知测量结论。

## 8. 学习提醒

- 默认关闭。
- 用户首次完成任务后，展示一次非强制开启提示。
- 默认提醒时间为每天上午 10:00，可在设置中修改。
- Android 13+ 仅在用户开启提醒时请求通知权限。
- 使用 WorkManager 安排唯一的本地提醒任务；系统省电策略可能造成小幅延迟，不申请
  精确闹钟特殊权限。
- 当天任务已经完成时不发送提醒；通知点击后进入“学习”首页。

## 9. 异常处理

- 内置音频损坏、TTS 初始化延迟、缺少英语 voice 和播放中断不能静默失败。
- 连续快速滑卡时只允许当前卡发音。
- 屏幕旋转、进程重启和任务退出不能重复记录同一次答题。
- Room 迁移失败时保留原数据库，不使用破坏式重建。
- 用户拒绝通知权限时只关闭提醒，不影响学习和音频。
- 候选中文不足或重复时，选项生成器跨当前词库补充并去重。

## 10. 测试与验收

### 单元测试

- TTS 初始化前的请求会在初始化完成后补播。
- 连续请求只播放最新词条，内置音频错误会回退 TTS。
- 两遍学习阶段顺序、退出恢复和末尾强化。
- 首次答案只更新一次 SM-2。
- SM-2 质量 2/5、最低 EF、1/6 天间隔和 21 天掌握边界。
- 错题权重增加、降低和退出错题入口。
- 7/30 天统计、未来到期数量和保持率公式。
- 上午 10:00 的下一次提醒时间计算。

### Room 与 Compose 测试

- V1 到 V2 迁移保留进度和收藏。
- 会话持久化、结束清理和跨日恢复。
- 首页两栏导航、底部大按钮和必要信息。
- 浏览卡片滑动、展开、发音、收藏。
- 新词巩固选项、答错反馈和末尾重现。
- 数据详情与提醒设置。

### 设备验收

- API 29 与 API 36 安装、升级和进程重启。
- 断网完成学习、复习、测试、统计和发音。
- 禁用系统英语 TTS 后，内置音频仍然可以播放。
- 删除一项测试音频后，TTS 能够自动接管。
- 小屏、大字号和手势导航下，开始按钮保持在拇指可达区域且不遮挡内容。

## 11. 本次范围外

V2 不加入账号、云同步、联网内容、社交、排行榜、成就系统、应用商店发布或多设备
同步。
